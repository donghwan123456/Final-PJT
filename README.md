## 페이지별 구현 기능

### 메인 페이지
- 캐러셀 구현, 사이드바 -> 각 페이지로 이동 가능


### 회원 가입, 로그인, 로그아웃
- 기본적인 회원 관리
### 프로필 페이지
- 회원 정보 수정, 목표자산, 가입상품 확인 가능
### 예적금 금리 비교
- 은행, 상품 종류에 따른 금융 상품 검색 기능

- 가입하기 버튼을 통해 자신이 가입한 상품 DB에 저장할 수 있음

### 주변 은행 찾기
- 사용자 위치 기반으로 주변 은행 검색 기능

### 게시판
- 게시글 등록, 수정, 삭제

- 댓글 등록, 수정, 삭제

- 작성자 본인 외 수정, 삭제 불가


### 주요 알고리즘 (맞춤형 상품 추천 기능)

- 설문조사에 4가지 질문을 배치
- 각 답변에 1에서 5점 사이의 점수를 부여
- 사용자가 설문조사 완료할 시 사용자 정보와 점수를 DB에 저장
- 금융감독원 API를 통해 상품 정보 가져올 때 점수대별로 추천 상품 다르게
- ex) 15점 초과 시 예금, 적금 정보 외에도 파생상품, 주식, 코인 등으로의 안내 가능하게

## 배운 점 & 아쉬운 점
- 윤동환

    - 과목마다 따로 따로 배웠던 기술들 프로젝트 구현 위해 동시에 사용해 보면서 각 기술들 간 어떤 부분에서 충돌이 발생하는지 알 수 있었음.

    - 코드의 재사용성, 가독성 등 여태껏 체감하지 못했던 사항 들을 직접 느끼며 중요성 깨달았음

    - 백엔드와 프론트엔드의 역할 구분이 아닌 페이지별 역할 구분을 통해 진행했는데 왜 다른 팀들이 프론트엔드와 백엔드로 역할 구분을 해서 프로젝트를 진행하는지, 왜 그런 방식이 효율적인지 알게 되었음

- 김지환
    - Vue, React 등의 프론트엔드 framework를 사용하지 않아서 SPA가 불가능하여 재사용성이 떨어지고 페이지 새로고침이 자주 발생함 -> 규모가 큰 프로젝트의 경우 무조건 프론트 프레임워크를 사용해야함을 깨달음

    - 팀원 역할 구분을 프론트와 백이 아닌 기능 별로 구분을 하여 진행해서 진행 순서가 매끄럽지 못하였음

    - 적절한 코드를 찾고 가져오는 것도 실력이라는 점을 깨달았음. 규모가 큰 프로젝트일수록 이 능력이 더 중요해질거 같음
